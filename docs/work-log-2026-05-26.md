## 반영 내용

### 1. DB 스키마 실행 스크립트 추가

- `tools/apply-sales-inventory-schema.mjs` 추가
- `schema_sales_inventory.sql` 전체를 PostgreSQL/Supabase에 실행하는 스크립트
- 실행 명령:

```bash
npm run db:schema
```

- 필요 환경변수:

```env
DATABASE_URL=Supabase PostgreSQL connection string
```

### 2. 엑셀 초기 데이터 반영 스크립트 추가

- `tools/seed-sales-inventory-from-excel.mjs` 추가
- 아래 엑셀 파일을 읽어 FN OS DB에 반영하도록 구성

```text
C:/Users/pains/Downloads/FN_거래처.xlsx
C:/Users/pains/Downloads/FN_창고.xlsx
C:/Users/pains/Downloads/FN_품목(제품).xlsx
```

- 반영 대상 테이블:

```text
customers
warehouses
products
```

- 실행 명령:

```bash
npm run db:seed:sales-inventory
```

- 필요 환경변수:

```env
SUPABASE_URL=FN OS Supabase project URL
SUPABASE_SERVICE_ROLE_KEY=FN OS Supabase service role key
```

### 3. 공통 env 로더 추가

- `tools/env-utils.mjs` 추가
- `.env.local`, `.env`를 읽어 DB 스크립트에서 사용
- 민감값은 출력하지 않음

### 4. 패키지/명령 추가

- `pg` 패키지 추가
- `package.json`에 DB 명령 추가

```json
"db:schema": "node tools/apply-sales-inventory-schema.mjs",
"db:seed:sales-inventory": "node tools/seed-sales-inventory-from-excel.mjs"
```

### 5. `.gitignore` 정리

- Vercel env pull 과정에서 중복으로 붙은 `.env*` 항목 정리
- `.env*` ignore 규칙은 기존 위치에 유지


### 엑셀 시드

```bash
npm run db:seed:sales-inventory
```

결과: 로컬 `SUPABASE_URL` 또는 `SUPABASE_SERVICE_ROLE_KEY` 값이 비어 있어 실행 보류

## 확인된 이슈

- `vercel env pull`로 내려받은 `.env.local`은 민감 환경변수 값이 빈 문자열로 내려온 상태
- 따라서 현재 로컬에서는 새 FN OS Supabase DB에 직접 접속할 수 없음
- Vercel에는 환경변수가 등록되어 있어도 로컬 자동 실행에는 별도 실제 값이 필요함
