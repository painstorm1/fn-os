# 2026-05-21 FN OS 작업 로그

## 요약

- 수입관리 첨부파일함 UI와 업로드 흐름을 정리했다.
- 매출/재고의 온라인 발주 작업실, 송장출력용/이카운트 입력 시트, 직송파일, 송장번호 매칭 흐름을 계속 보강했다.
- 이카운트 API 연동 준비와 품목검색 UX를 개선했다.
- Google Sheets 기반 FN_택배시트 반영 기능을 연결했다.
- 버튼 피드백, 좌우 패널 높이, 단축키, 파일 아이콘 등 FN OS 전체 사용감을 다듬었다.

## 수입관리

- 발주별 첨부파일함을 정리했다.
  - 첨부파일 제목을 `첨부파일`로 단순화했다.
  - 파일명 클릭과 열기 버튼이 같은 동작을 하도록 정리했다.
  - 문서 유형 선택/표시는 제거하고 파일명, 크기, 업로드일, 메모, 작업만 남겼다.
  - 파일 확장자는 텍스트 태그 대신 아이콘형 표시로 바꿨다.
- 첨부파일 업로드 오류를 추적했다.
  - Supabase Storage RLS/Service Role Key 문제를 확인했다.
  - 기존 수입ERP API 서버 쪽 Vercel 환경변수에 `SUPABASE_SERVICE_ROLE_KEY`가 필요하다는 점을 정리했다.
- 엑셀 첨부파일 열기 흐름을 Google Sheets 기반으로 바꾸는 방향을 적용했다.
- 발주 리스트의 폴더 아이콘 위치와 파일 개수 표시를 조정했다.
- 빠른수정 화면에서 송장매칭 실패 행 표시처럼, 실패 지점을 화면에서 직접 찾기 쉬운 방식으로 바꿨다.
- 부자재만 있는 주문의 예상원가 표시 규칙을 정리했다.
  - 단일 부자재 주문은 `총금액 / 수량` 기준 예상원가를 총비용 카드에 표시하는 방향.
  - 단가가 다른 여러 부자재 주문은 옵션별 원가/마진표에 옵션명, 수량, 예상원가만 표시하는 방향.

## 매출/재고 - 온라인 발주

- 기존 발주통합매크로 흐름을 FN OS 화면으로 옮기는 작업을 계속했다.
- 파일 업로드 영역을 다듬었다.
  - 발주파일/송장파일 업로드를 분리했다.
  - 파일 타입별 배지를 붙였다: 이카운트 `E`, 오늘의집 `O`, 토스 `T`, 현대이지웰 `Z`, 송장파일 `CJ`.
  - 중복 파일명은 다시 업로드되지 않도록 방지하는 방향으로 정리했다.
  - 업로드 파일은 새로고침/메뉴 이동 후에도 작업 초기화 전까지 유지되도록 했다.
- F1~F6 단축키와 버튼명을 정리했다.
  - `F1. 발주 작업 실행`
  - `F2. 송장출력용 엑셀`
  - `F3. 직송파일 생성`
  - `F4. EC_판매입력 전송`
  - `F5. 송장번호 매칭`
  - `F6. FN_택배시트 반영`
- 송장출력용 시트 변환 규칙을 보강했다.
  - 주문옵션 텍스트 정리 규칙을 추가했다.
  - `[RG]`, `[NG]` 같은 접두 태그는 제거했다.
  - 연락처1/연락처2 중 하나가 비어 있거나 `-`, `--`이면 다른 값으로 보완하게 했다.
  - A열 주문코드 넘버링에 오전/오후 구분을 반영했다.
- 정렬 UX를 개선했다.
  - 송장출력용, 이카운트_송장입력, 이카운트_판매입력에서 헤더 더블클릭 시 오름차순/내림차순 반복 정렬.
  - 정렬 기준은 숫자, 영어, 한글 순서에 맞추는 방향으로 조정했다.
- 전체 엑셀 내보내기를 보강했다.
  - 파일명 규칙을 `MMDD_FNOS통합발주`로 정리했다.
  - 전체 폰트는 맑은 고딕 11로 맞추는 방향으로 수정했다.
  - A열/G열 등 주요 열 너비 규칙을 정리했다.
  - J열 정산예정금액이 엑셀에서 숫자처럼 보이도록 처리했다.
- 직송파일 생성 흐름을 개선했다.
  - 거래처 선택을 입력창이 아니라 `JB`, `케이모아` 고정 선택 UI로 바꿨다.
  - 선택된 거래처별로 서로 다른 엑셀 양식이 나오도록 정리했다.
  - Shift/Ctrl 기반 다중 행 선택을 보강했다.
  - 동일 데이터 중복 입력 시 확인/방지하는 방향으로 수정했다.

## 송장번호 매칭

- 송장파일 업로드는 발주파일 작업과 별도 흐름으로 다루도록 정리했다.
- 매칭 기준을 정리했다.
  - 송장파일의 받는분, 전화번호, 주소를 정규화해서 송장출력용 행과 비교.
  - 이름, 전화번호, 주소의 공백/하이픈 차이를 최대한 흡수.
  - 송장파일의 내품수량이 2 이상이면 해당 행을 수량만큼 확장해서 매칭.
- 매칭 실패 표시를 개선했다.
  - 실패 메시지는 일반 팝업으로 표시.
  - 실패한 송장출력용/이카운트_송장입력 행은 옅은 노란색으로 표시.
  - 실패가 없으면 `송장매칭 성공` 팝업 표시.
- 실패 안내는 복사 가능한 별도 모달보다 단순 팝업 + 행 하이라이트 중심으로 정리했다.

## 이카운트 API / 매출 DB

- 이카운트 API 연동 준비를 진행했다.
  - 사용 가능 API: 판매입력, 품목조회, 재고현황, 구매입력.
  - 환경변수와 테스트/실서버 전환값을 정리했다.
  - `ECOUNT_USE_TEST_SERVER`는 테스트 서버 사용 여부를 제어하는 값으로 안내했다.
- 이카운트 로그인 세션 파싱 오류를 수정했다.
- 품목검색 경로 오류를 수정했다.
- 이카운트_판매입력 시트의 품목코드 칸에서 검색어 입력 후 엔터 시 품목검색 창을 띄우는 흐름을 추가했다.
- 품목검색 팝업의 색상 방향을 FN OS 브랜드에 맞춰 검정/오렌지 중심으로 바꾸는 방향을 잡았다.
- EC_판매입력은 먼저 FN OS DB에 저장하고, 이후 이카운트 판매입력 API로 전송하는 구조로 정리했다.
- 날짜별/거래처별/품목별 매출 조회가 가능하도록 sales DB 기반 흐름을 준비했다.

## Google Sheets

- `FN_택배시트` 반영을 Google Sheets API로 처리하는 방향으로 연결했다.
- Vercel 환경변수로 Google Sheets 관련 값이 들어갔고, 서비스 계정 공유 권한이 필요하다는 점을 확인했다.
- 시트 범위 파싱 오류와 문서 타입 오류를 추적했다.
- 동일 데이터가 이미 있을 경우 경고하도록 방향을 잡았다.

## UI / 사용감

- FN OS 전체 버튼에 클릭 피드백을 추가했다.
- 매출/재고 메뉴 진입 후 다른 메뉴 이동이 막히던 문제를 확인하고 수정했다.
- 로고 클릭으로 메인 대시보드 복귀가 매출/재고 화면에서도 동작하도록 복구했다.
- 오른쪽 패널과 왼쪽 패널이 별도 내부 스크롤을 만들지 않고 페이지 높이에 자연스럽게 맞춰지도록 조정했다.
- 매출/재고 상단 KPI/준비 안내 영역은 제거하고, 온라인 발주 작업 영역 중심으로 정리했다.
- 오른쪽 패널은 기본 상태에서 상품 간편조회만 열리고, 간편 등록/간편 입력은 닫힌 상태가 되도록 정리했다.
- 수입관리 발주/제품 화면에서만 F2 단축키를 적용했다.
  - 발주 화면: `F2 +새 발주`
  - 제품 화면: `F2 +새 제품`

## 확인된 주의사항

- `.env.example`은 기존 dirty 상태로 남아 있으며, 오늘 작업 로그 커밋에는 포함하지 않는다.
- 수입관리 첨부파일 업로드는 기존 수입ERP API 서버의 Supabase Service Role Key 설정과 Storage 정책 영향을 받는다.
- 이카운트 API는 세션 발급, API 인증키, 허용 IP, 테스트/실서버 여부에 따라 결과가 달라질 수 있다.
- Google Sheets 연동은 서비스 계정 이메일이 대상 구글시트에 편집자로 공유되어 있어야 한다.
- Vercel 환경변수를 추가한 뒤에는 배포가 새로 반영되어야 한다.
