## 오늘 반영된 주요 수정 확인

### 1. 주문파일 암호 처리

- `ORDER_FILE_PASSWORD` 환경변수 사용 경로 확인
- Vercel 환경변수가 없거나 암호 오류가 발생하면 비밀번호 입력 팝업으로 재시도하는 흐름 확인
- 오늘의집 암호화 엑셀 대응 코드 경로 확인

상태: 코드상 정상

주의:
- Vercel에 환경변수를 추가한 뒤에는 새 배포가 완료되어야 적용됨

### 2. 업로드 파일 출처 분류

아래 파일명 패턴 분류 코드 확인:

- `ESK003M` → 이카운트 주문수집
- `배송목록...` → 현대 이지웰
- `주문배송 내역...` → 오늘의 집
- `주문배송관리-상품준비중...` → 토스

알 수 없는 파일은 사용자 확인 팝업이 뜨도록 되어 있음.

상태: 코드상 정상

### 3. 송장출력용 주문옵션 매핑

이카운트 주문수집의 `주문옵션` 값은 아래 우선순위로 처리되도록 확인:

1. `품목명(ERP)`
2. `품목명`
3. `주문옵션`
4. `쇼핑몰상품명`

상태: 오늘 수정 완료

### 4. 쿠팡 정산예정금액 0.88 반영

이카운트 주문수집 중 쿠팡으로 분류되는 행만:

```text
정산예정금액 = 원금액 * 0.88
```

으로 계산되도록 확인.

영향 범위:

- 송장출력용 정산예정금액
- 이카운트 판매입력 단가/공급가액 계산 기준

상태: 오늘 수정 완료

### 5. 주문옵션 정렬

정렬 기준 확인:

- `[RG]`, `[NG]` 같은 대괄호 prefix 제거 후 비교
- 한글 기반 오름차순 유지
- 같은 위치에서 영문 토큰과 한글 토큰이 충돌하면 영문 토큰 우선
- 예: `나이키 ED...`가 `나이키 에센셜...`보다 먼저 정렬

상태: 코드상 정상

### 6. 송장출력용 그리드

확인한 기능:

- 클릭: 셀 선택
- 더블클릭: 셀 편집
- 드래그 선택
- 붙여넣기
- 헤더 더블클릭 정렬
- 헤더 재더블클릭 시 반대 방향 정렬
- 행 높이 기본 30
- 송장출력용 열 너비는 기본 95px, `주문옵션`만 내용 길이 기반 자동 확장

상태: 코드상 정상

### 7. 직송파일 만들기

확인한 기능:

- 버튼 클릭 시 거래처 선택 모달 표시
- 선택 항목: `JB`, `케이모아`
- 직접 입력 prompt 방식 제거
- 선택한 행만 새 엑셀로 생성
- 같은 값이면 중복 추가 방지
- JB/케이모아 별도 헤더/고정값 매핑 적용

상태: 코드상 정상

### 8. 송장번호 매칭

확인한 기능:

- 송장파일이 없으면 실행 전 알림
- 송장출력용 데이터가 없으면 실행 전 알림
- 직접 입력 대상이 없으면 메모장 표시 생략
- 직접 입력 대상이 있으면 화면 내부 메모 영역 표시

상태: 코드상 정상

### 9. FN_택배시트 반영

확인한 기능:

- 송장출력용 데이터가 없으면 실행 차단
- 중복 의심 행이 있으면 확인 팝업
- 이미 실행한 작업이면 확인 팝업
- 실제 구글 시트 직접 쓰기는 브라우저 보안상 현재는 클립보드/텍스트 대체 흐름

상태: 현재 구조에서 정상

주의:
- 구글 시트 직접 쓰기는 Google API 서비스 계정 방식으로 별도 서버 API가 필요함

### 10. 수입관리 발주 폴더 첨부파일

FN OS UI:

- 문서 유형 드롭다운 제거
- 파일 선택/메모/업로드만 유지
- 첨부 목록에서 문서 유형 컬럼 제거
- 드래그앤드랍 다중 파일 UI 유지

수입ERP API:

- `/api/fnos/orders/<order_id>/attachments`
- `/api/fnos/attachments/<attachment_id>`
- `doc_type` 없어도 업로드 가능
- 기존 `doc_type` DB 컬럼은 유지
- Supabase Storage 키 이름 호환:
  - `SUPABASE_SERVICE_ROLE_KEY`
  - `SUPABASE_KEY`
  - `SUPABASE_ANON_KEY`
- Supabase Storage 업로드를 `POST /storage/v1/object/...` 방식으로 변경
- 파일 경로 URL 인코딩 적용
- 실패 시 Supabase 상세 오류가 화면에 나오도록 변경

상태: 코드상 보강 완료

주의:
- 실제 업로드 성공 여부는 `fn-import-erp` Vercel 최신 배포 완료 후 다시 확인 필요
- 현재 화면에서 계속 실패한다면 새 오류 메시지 본문을 기준으로 추가 수정 필요

## 속도/안정성 점검

### 빌드 성능

- FN OS production build는 약 5초대로 정상 범위
- 라우트 수가 많지 않아 빌드 병목은 크지 않음

### 화면 속도 관련

현재 구조에서 속도에 민감한 부분:

- `src/app/page.tsx`에 수입관리/매출재고 UI가 많이 모여 있음
- 제품 목록은 sessionStorage 캐시가 일부 적용되어 있음
- 매출/재고 엑셀 그리드는 행/열 상태가 많아 데이터가 많아질수록 렌더 부담 가능

보수적 권장:

- 당장 잘 되는 기능은 유지
- 다음 안정화 때 `page.tsx`를 기능별 컴포넌트로 분리
- 엑셀 그리드는 대량 행 기준으로 가상 스크롤 검토
- 첨부파일 업로드는 배포 후 실제 오류 메시지를 보고 추가 조정

## 발견된 잔여 이슈

### 1. `npm run lint` 실패

실패 원인:

- React 19/Next 16 lint 규칙에서 `useEffect` 내부 즉시 `setState` 패턴을 오류로 판단
- `<img>` 사용 경고
- 일부 unused 변수 경고

판단:

- production build는 통과
- lint 실패는 오늘 한 기능 하나의 즉시 장애라기보다 기존 대형 단일 파일 구조에서 생긴 누적 이슈
- 지금 대규모로 고치면 안정된 화면을 흔들 수 있으므로 오늘은 수정하지 않음

권장:

- 별도 리팩터링 작업으로 처리

### 2. 수입ERP 작업트리 dirty

현재 수입ERP 저장소에는 과거부터 남아 있는 변경/삭제 표시가 있음.

오늘 커밋에는 `app.py`만 포함했고, 기존 dirty 파일은 건드리지 않았음.

주의:

- 다음 수입ERP 작업 전에는 이 dirty 항목을 별도 정리 여부 결정 필요
- 특히 `database.db`, `static/css/app.css`, `vercel.json`, `수입ERP_` 삭제 표시가 남아 있음
