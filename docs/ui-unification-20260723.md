## 2. 통일 원칙

1. **같은 의미는 같은 위치와 표현**
   - 페이지 제목·설명·대표 작업은 `PageHeader`.
   - 카드 내부 제목·보조 작업은 `SectionHeader`.
   - 검색·기간·상태 필터는 `FilterBar`.
   - 저장·추가·수정·삭제는 공통 버튼 변형을 사용한다.
2. **업무별 고밀도 표는 유지**
   - 주문수집, 송장, 판매·구매 입력처럼 엑셀형 조작이 필요한 화면을 일반 카드형 목록으로 단순화하지 않는다.
   - 열 너비, 범위 선택, 복사·붙여넣기, F1~F4 단축키는 업무 기능으로 보존한다.
3. **탭·필터·상태를 구분**
   - 화면 자체를 바꾸면 탭, 현재 데이터만 좁히면 필터, 실행 결과는 알림으로 표현한다.
4. **모달은 한 규칙**
   - `role=dialog`, `aria-modal`, 제목 연결, 최초 포커스, Tab 포커스 고정, Escape 닫기, 닫힌 뒤 원래 위치로 포커스 복귀, 배경 스크롤 잠금을 공통 뿌리에서 처리한다.
5. **URL은 현재 작업 위치의 정본**
   - 메뉴·하위 탭·기간처럼 다시 열어야 하는 상태는 URL에 유지한다. 단순 UI 열림/닫힘은 로컬 상태로 둔다.
6. **모바일에서 기능이 사라지지 않게 함**
   - 데스크톱 사이드바를 단순히 숨기지 않고 메뉴 드로어 진입점을 제공한다.
7. **전역 재설계보다 기존 패턴 재사용**
   - 신규 UI 라이브러리를 추가하지 않는다. `fn-ui.tsx`, 기존 CSS 토큰, 현재 Next/React만 사용한다.

## 3. 메뉴별 차이와 통합 방향

### 공통 앱 셸

**현재**
- 데스크톱 왼쪽 메뉴는 `lg` 미만에서 완전히 숨지만 대체 진입점이 없다.
- 오른쪽 도구 패널은 `xl` 미만에서 사라진다.
- 상위 메뉴·하위 메뉴 버튼의 펼침 상태와 현재 위치에 `aria-current`, `aria-expanded`가 없다.
- 일부 내부 이동은 `Link`, 일부는 `window.location.href`, 일부는 `history.replaceState`를 사용한다.

**통합**
- 모바일 메뉴 버튼과 오버레이 드로어를 추가한다.
- 상단 공통 바에서 현재 대메뉴·하위 메뉴 경로를 보여준다.
- 링크의 새 탭/새 창 기본 동작을 보존하고, 버튼 기반 내부 이동만 공통 이동 helper를 사용한다.
- 현재 메뉴와 펼침 상태를 보조기기에 전달한다.

### 대시보드

**잘된 점**
- 매출·비용·재고·최근 거래를 첫 화면에서 파악할 수 있다.
- API 캐시를 이용해 재진입 시 체감 로딩을 줄인다.

**현재 차이/불편**
- 다른 핵심 메뉴와 달리 `PageHeader`가 없다.
- 캐시가 없을 때 로딩 표현이 독자적이고, 조회 실패가 카드 일부에 섞인다.

**통합**
- 공통 페이지 헤더와 설명을 적용한다.
- 공통 로딩·오류·빈 상태 표현을 사용한다.
- 지표 카드와 차트의 업무 전용 구성은 유지한다.

### 매출/재고

**잘된 점**
- 온라인 주문, 판매/구매 이력, 재고, 기초정보의 업무 흐름이 강하다.
- 범위 선택·키보드 이동·엑셀 업로드·F1~F4 단축키처럼 반복 업무 속도를 높이는 기능이 있다.

**현재 차이/불편**
- 상태 선택 버튼, 실행 버튼, 보조 내보내기 버튼의 높이·색·위험 표현이 같은 화면 안에서도 다르다.
- 송장 엑셀/FN판매입력/FN구매입력은 CSS 체크박스로 여는 독자 모달이라 공통 Escape·포커스·접근성 규칙을 타지 않는다.
- 성공/실패가 고정 토스트, 본문 메시지, 브라우저 alert로 나뉜다.
- 일부 표의 더블클릭 기능은 안내가 부족하다.

**통합**
- 상태는 상태 필터, 업무 실행은 공통 액션 버튼, 위험 작업은 danger 변형으로 의미를 분리한다.
- 3개 대형 시트 팝업을 공통 `SelectionModal` 흐름으로 옮긴다.
- 페이지 알림 영역을 공통 live-region 규칙으로 맞춘다.
- 범위 선택·F1~F4·엑셀형 표 밀도는 유지한다.

### 수입관리

**잘된 점**
- 발주·제품·설정이 URL 경로로 구분되고 각 화면에 설명이 있다.
- 발주 상세/수정/첨부/제품 연결 등 실제 업무 단계가 명확하다.

**현재 차이/불편**
- 리스트·상세·폼의 뒤로가기/저장/삭제 배치가 화면마다 다르다.
- 일부 로딩 분기는 빈 패널 또는 `null`을 보여준다.
- 첨부·발주·제품 팝업은 공통 모달을 쓰지만 공통 모달 자체의 포커스 처리가 부족하다.

**통합**
- 공통 모달 뿌리 수정으로 모든 발주·제품·첨부 팝업을 한 번에 개선한다.
- 빈 로딩 분기를 공통 로딩 상태로 바꾼다.
- 상세/폼의 업무 데이터와 단계 표시 방식은 유지한다.

### 광고분석

**잘된 점**
- 비용·구매전환수·ROAS를 포함한 핵심 광고 지표와 채널 비교가 명확하다.
- 날짜 범위가 URL과 연결돼 재진입·공유가 가능하다.

**현재 차이/불편**
- 날짜·파일 업로드 도구가 오른쪽 패널에만 있어 작은 화면에서 사라진다.
- 오류는 카드, 업로드 결과는 문자열, 일부 작업은 별도 확인 모달을 사용한다.
- 채널 선택 체크박스의 포커스/크기가 다른 폼과 다르다.

**통합**
- 공통 상단 경로와 컨트롤 포커스 규칙을 적용한다.
- 업로드·오류·교체 확인은 공통 알림/모달 규칙을 사용한다.
- 광고 전용 지표 카드와 비교표는 유지한다.

### 회계/비용

**잘된 점**
- 대시보드, 원장, 고정비, DB 검토가 업무 목적별로 잘 나뉜다.
- 기간 이동, 통장/카드 구분, 카테고리 필터가 촘촘하다.

**현재 차이/불편**
- 통장/카드 전환은 탭처럼 보이지만 독자 버튼 그룹이고, 기간 이동/필터/엑셀 버튼 높이가 섞인다.
- 일부 조회 실패·로켓그로스 결과·포인트 결과의 메시지 모양이 다르다.
- 표 페이지 이동 규칙이 매출/재고와 다르다.

**통합**
- 통장/카드 전환은 공통 탭, 나머지는 공통 필터 바 의미로 정리한다.
- 결과 메시지에 공통 상태 톤과 live-region을 적용한다.
- 금액 계산·원장 분류·기간 기준은 변경하지 않는다.

### 자동화센터

**잘된 점**
- 실행 기록, 상태, 신규 작업 등록·수정·즉시 실행 흐름이 한 화면에 있다.
- 이미 `PageHeader`, `SectionHeader`, `FormModal`을 적극 사용한다.

**현재 차이/불편**
- 카드 내부 일부 버튼이 raw 버튼이라 페이지 상단 액션과 밀도가 다르다.
- 로딩/오류/실행 결과 메시지가 다른 메뉴의 표현과 다르다.

**통합**
- 남은 독자 버튼·입력·상태 메시지를 공통 컴포넌트로 옮긴다.
- 자동화 실행 API와 상태 로직은 유지한다.

### 설정

**잘된 점**
- 회사정보·계정·장소·비밀번호 영역이 분리되어 있다.
- 추가/수정 작업 대부분이 공통 모달을 사용한다.

**현재 차이/불편**
- 상위 탭이 독자 버튼 그룹이고 회사/계정/장소의 목록 작업 규칙이 조금씩 다르다.
- 인사관리 재인증 정보가 브라우저 저장소에 있어 UI 숨김과 실제 권한 경계가 혼동될 수 있다.

**통합**
- 상위 전환을 공통 탭으로 통일하고 목록/모달 버튼 순서를 맞춘다.
- 브라우저 저장소 기반 인사관리 권한은 단순 UI 수정으로 안전하게 해결할 수 없는 별도 서버 권한 과제로 명시한다. 이번 UI 브랜치에서 보안을 약화시키는 우회는 추가하지 않는다.

### 로그인/첨부 미리보기

**현재 차이/불편**
- 로그인 입력·버튼·오류 표시가 앱 내부 공통 컴포넌트와 분리되어 있다.
- 첨부 미리보기는 독립 라우트라 앱 셸과 다르지만, 파일 확인 목적상 전체 셸을 강제할 필요는 없다.

**통합**
- 로그인 입력·버튼·오류 live-region은 공통 규칙을 사용한다.
- 첨부 미리보기는 독립 작업 화면을 유지하되 포커스와 버튼 상태만 공통 규칙에 맞춘다.

1. 데스크톱에서 왼쪽 메뉴의 5개 대메뉴와 모든 하위 메뉴를 차례로 연다.

3. 창 폭을 줄여 모바일 메뉴 버튼으로 모든 대메뉴에 진입할 수 있는지 본다.
4. 수입관리에서 발주 상세·수정·첨부 모달을 열고 Tab/Shift+Tab/Escape/닫은 뒤 포커스 복귀를 확인한다.
5. 매출/재고에서 송장 엑셀, FN판매입력, FN구매입력 대형 팝업을 열어 범위 선택·내보내기·닫기를 확인한다.
6. 매출/재고의 F1~F4와 엑셀 업로드/선택 기능이 기존과 동일한지 확인한다.
7. 광고분석에서 날짜 범위 이동, 채널 선택, 파일 대기/교체 확인을 확인한다.
8. 회계/비용에서 통장/카드 전환, 월/일 기간 이동, 엑셀 다운로드를 확인한다.
9. 자동화센터에서 목록 새로고침, 작업 등록/수정, 실행 버튼의 상태를 확인한다.
10. 설정에서 회사/계정/장소 탭 및 추가·수정 모달을 확인한다.
